# Incident Readiness Checklist

## Detection

- [ ] Critical alerts map to user/business impact
- [ ] Alert thresholds are reviewed
- [ ] Noise and false positives are tracked
- [ ] Unknown failure modes are captured after incidents

## Response

- [ ] On-call owner is clear
- [ ] Escalation path is documented
- [ ] Runbooks exist for critical flows
- [ ] Communication channel is defined

## Recovery

- [ ] Rollback path is known
- [ ] Data recovery procedure exists
- [ ] Provider failure plan exists
- [ ] Manual workaround is documented where needed

## Learning

- [ ] Postmortem template exists
- [ ] Action items have owners
- [ ] Recurring incident patterns are tracked
- [ ] Preventive controls are added to roadmap
